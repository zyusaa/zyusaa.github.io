1、在本地计算机安装Hexo环境
1.1 安装Node.js
1.2 安装Git
1.3 安装Hexo

在D盘新建MyHexoBlogs文件夹用来存放个人博客，进入该文件夹，右键打开Git Bash，使用 npm 安装 Hexo。

安装太卡

Hexo安装完成

Hexo安装完成后，在MyHexoBlogs文件夹下新建myblogs项目，并对其进行初始化。

Hexo init myblog
Cd myblog
Npm install

此时，会在MyHexoBlogs文件夹下新建myblogs文件夹，并在其内部生成相应的项目文件。如下图所示：

在myblogs文件夹下启动hexo服务。
输入 hexo g  ，Hexo server

此时在本地打开浏览器，通过 http://localhost:4000/ 便可访问基于Hexo的个人博客主页了。如下图所示：

二、在GitHub创建一个仓库 https://github.com/zyusaa/zyusaa.github.io.git
三、Hexo部署到github
1.生成ssh密钥
右键单击鼠标，点击 Git Bash Here输入以下命令：
git config –global user.name “Name”
git config –global user.email “Email”
Name和Email是我们注册Github时的用户名和邮箱。
然后生成密钥：
ssh-keygen -t rsa -C “Email”
Email是我们注册Github时的邮箱

//到这里可以直接回车将密钥按默认文件进行存储

//这里是要你输入密码，其实不需要输什么密码，直接回车就行

现在密钥已经生成，一般存放在（/c/Users/you/.ssh/id_rsa.pub.）
我们运行下面的命令将密钥复制为粘贴板
clip < ~/.ssh/id_rsa.pub

2.连接GitHub

在Github头像下面点击Settings，再点击SSH and GPG keys，new一个SSH，名字任意。然后将刚才复制的密钥添加就可以了

这里需要在本地进行验证一次
输入命令：ssh -T git@github.com 并且输入yes之后，行末尾会显示你的用户名

用记事本打开博客根目录下的_config.yml文件，这是博客的配置文件，我们需要修改一下才能连接Github。

repository修改为你自己的github项目地址。(repository属性的地址就是之前创建仓库后所保存的http地址)
每一个冒号后面都有一个空格。

四、写文章、发布文章
1.创建扩展
首先在博客根目录下右键打开Git Bash Here输入以下命令，安装一个扩展.
npm i hexo-deployer-git

2.然后输入下面的命令，新建一篇文章。
hexo new post “article title”

新建文章之后需要生成一遍文件
输入命令：hexo g

新建文章后可在本地先预览一遍
输入命令：hexo s浏览器输入：localhost:4000(ctrl + c 关闭)

Ps：4000端口被占用解决办法↓
在cmd窗口找到占用端口的进程终止它

最后直接部署到Github就可以啦
输入命令：hexo d
然后打开C:\panakot Blog\Hexo\source_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章文件啦。
你的博客地址：https://你的用户名.github.io，现在每个人都可以通过此链接访问你的博客
